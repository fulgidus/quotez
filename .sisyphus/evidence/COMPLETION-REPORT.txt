## ACCUMULATED WISDOM (FROM NOTEPAD)

### Zig 0.16 API Patterns
1. **Networking**: `std.net` removed → use `std.posix` directly
   - `socket()`, `bind()`, `listen()`, `accept()`, `sendto()`, `recvfrom()`
   - `setsockopt()` for SO.REUSEADDR
   - `fcntl()` with F.SETFL and O.NONBLOCK

2. **Time API**: `std.time.Instant.now()` with `.timestamp.sec` and `.nsec`
   - `nanoTimestamp()` may not exist (not verified)

3. **File Operations**: `std.fs.Dir.statFile()` returns `std.fs.File.Stat`
   - `mtime` field: `.sec` and `.nsec` (i128 total nanoseconds)

4. **Error Handling**: `std.posix` functions return standard error unions
   - Map POSIX error sets appropriately

### Project Conventions
1. **Logging**: Use structured logging with contextual fields
   - Example: `log.info("hot_reload_complete", .{ .quotes = count });`

2. **Memory Management**: Careful with allocator ownership
   - Use `errdefer` for cleanup in init functions
   - Explicit `deinit()` methods for all structs with allocations

3. **Selection Modes**: All 4 modes operational
   - `random`: Fresh random per call
   - `sequential`: Linear iteration with wrap-around
   - `random_no_repeat`: Random with exclusion list
   - `shuffle_cycle`: Shuffle deck, iterate, reshuffle

4. **Quote Parsing**: All 5 parsers working
   - txt, csv, json, toml, yaml
   - Deduplication via Blake3 content hashing
